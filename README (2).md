# Dashboard Pages

The dashboard is split into three pages, each answering a different question for a different audience.

## Page 1 — Executive Overview

**Audience:** C-suite, board members
**Question it answers:** *"How are we doing overall?"*

### Layout
- **Top row:** 4 KPI cards
  - Satisfaction Rate % (43.3%)
  - On-Time Arrival % (56.3%)
  - Overall Service Score (3.24 out of 5)
  - Loyal Customer Share % (81.7%)

- **Middle row:**
  - Donut chart — Satisfied vs Neutral/Dissatisfied with center label
  - Clustered column — Passengers by class & satisfaction

- **Bottom row:**
  - Horizontal bar — Average service ratings, sorted ascending with conditional color (red/amber/green)
  - Line chart — Satisfaction by age band (the "hump curve")

- **Left sidebar:** Slicers for Travel Type, Customer Type, Gender, Age Band

### Why this layout
Executive pages should answer big questions in 10 seconds. The KPIs establish the headline; the donut anchors it visually; the class chart reveals the biggest segmentation; the service ratings bar shows the operational pain points; the age line reveals the demographic story.

## Page 2 — Service Strategy

**Audience:** Operations / strategy team
**Question it answers:** *"Where should we invest our budget?"*

### Layout
- **Quadrant scatter chart** (top-left) — Rating vs Importance
  - X-axis: average rating (2.6 to 3.8)
  - Y-axis: correlation with satisfaction (-0.1 to 0.55)
  - 14 dots, one per service, with labels
  - Two dashed median lines divide the chart into 4 quadrants:
    - **INVEST** (low rating, high importance): Online Boarding, Wifi
    - **PROTECT** (high rating, high importance): Seat Comfort, Entertainment
    - **IGNORE** (low rating, low importance): Gate Location, Time Convenience
    - **OVER-INVESTED** (high rating, low importance): Baggage, Inflight Service

- **Satisfaction lift bar chart** (top-right)
  - Y-axis: 14 services
  - X-axis: lift (% satisfied at rating 5 minus % satisfied at rating 1)
  - Sorted descending — biggest levers at top
  - Shows the **effect size**, complementing the quadrant's positioning

- **Distance × Class heatmap** (bottom-left)
  - Matrix with red→yellow→green gradient
  - Rows: distance bands (Short, Medium, Long, Ultra-long)
  - Columns: Business, Eco, Eco Plus
  - Reveals that Business gets better with distance, Economy gets worse

- **Loyalty resilience line chart** (bottom-right)
  - Two lines: Loyal vs Disloyal customers
  - X-axis: delay buckets (On time, 1-15 min, 16-60 min, 60+ min)
  - Y-axis: Satisfaction Rate %
  - Shows the counterintuitive "loyal customers drop more" finding

### Why this layout
This is the most insight-dense page. Each visual answers a strategic question:
- Quadrant: *which services matter?*
- Lift: *how much do they matter?*
- Heatmap: *where does the product fail?*
- Resilience: *who do we lose first when ops fail?*

Note: This page **does NOT have slicers**, intentionally. The strategy shouldn't change based on user filtering — it should represent the structural truth of the data.

## Page 3 — Root Cause Analysis (Drill-through)

**Audience:** Analysts, product managers
**Question it answers:** *"Why is this specific segment underperforming?"*

### How to access
Right-click any bar / column / cell representing a class, travel type, customer type, or age band on Page 1 or Page 2 → Drill through → Root Cause Analysis.

### Layout
- **Header:** Dynamic title showing the drilled segment (e.g., "Root cause analysis — Eco")
- **Back button** (auto-generated by Power BI) returns to the previous page

- **Top row — 3 KPI cards:**
  - Segment Satisfaction % (the drilled segment's value)
  - Benchmark Satisfaction % (overall average for comparison)
  - Gap vs Benchmark (▲ green or ▼ red, percentage points)

- **Middle row:**
  - Service ratings bar — same format as Page 1, but auto-filtered to the segment
  - Travel × Age heatmap — reveals sub-segments within the drilled segment

- **Bottom row:**
  - Delay impact bar — satisfaction by delay bucket for this segment
  - % Bottom Box by service — % of passengers rating each service 1-2 stars

### Why this layout
Root cause analysis works best when comparison is built into the page. Every metric shows both the segment's value AND the overall benchmark. The user can immediately see "this is bad, AND it's worse than average."

The `USERELATIONSHIP` DAX pattern is essential here — without it, the service ratings and bottom box visuals wouldn't respond to the drill-through filter.

## Color System

Consistent across all three pages:
- **Green** `#1D9E75` — Satisfied / Good / Above benchmark
- **Red** `#E24B4A` — Dissatisfied / Critical / Below benchmark
- **Amber** `#EF9F27` — Warning / Mid-range (3.0–3.5 ratings)
- **Navy** `#1F4E79` — Brand color for titles
- **Gray** `#666666` — Secondary text, gridlines

Heatmaps use a continuous red → yellow → green gradient centered at the overall benchmark (~43%).
